Why

피드는 어디서나 똑같지만,
동네의 지금은 다릅니다

기존 SNS의 타임라인은 위치와 무관하게 흘러갑니다. Now Here는 위치가 곧 타임라인 — 대기줄, 골목 분위기, 동네 소식 같은 "현장의 지금"을 그 자리에서 공유하고, 그 자리에서 소비합니다.

Core · 두 개의 렌즈

같은 지도, 같은 컨텐츠 —
구획 단위만 바꿔 봅니다

베이직 모드행정동(洞) 기준. 서비스의 기본 지역 단위 — 내 동네의 일상과 소식.
트렌드 모드유저가 모이는 핫플 "트렌드존" 기준(관리자 큐레이션). 전환 시 근접 존 자동 뷰.

위치명 · 포커스 렌즈 · 피드 필터 · 동네 채팅방이 모드에 따라 동↔존으로 함께 전환되고, 존 밖에서는 동 이름으로 폴백됩니다.

Core · 몰입 장치

포커스 렌즈 🔍

보고 있는 구역(동/존)만 선명하게, 주변은 옅은 안개로. 확대하면 자동으로 켜지고, 구역을 탭하면 핀 고정 + 화면 맞춤. 렌즈 밖 스팟은 옅어지고, 소식 지면은 해당 동으로 슬라이드 — 강조가 컨텐츠를 방해하지 않게 설계했습니다.

Contents

현장을 담는 5가지 방법

  • 💬
    스팟 메시지 — 지도 위 말풍선으로 남기는 한 줄 일상. 계정 간 실시간 공유.
  • 📸
    라이브 카메라 — 찍으면 바로 피드에. 현 위치의 동 + 트렌드존 자동 태깅.
  • ❤️
    더블탭 좋아요 — 계정당 1개, 실시간 합산. 존 하트 랭킹과 베스트 썸네일의 원천.
  • 📍
    현장 Request — "지금 파이브가이즈 줄 길어요?" 원격 질문 → 현장 유저 퀵응답 알림.
  • 🗞️
    요약 지면 — 상단 카드 공간. 베이직=동네소식 카드, 트렌드=존 리스트.
Service Tabs

세 개의 소비 방식

🧭 지도스팟·Request 마커 + 포커스 렌즈. 공간 그대로의 탐색.
🖼 피드그리드(1~4열·핀치). 범위 필터가 모드를 따라감: 현재 동네/Trend Zone/전체(거리+최신).
👥 소셜현 위치명 동네 채팅방 + 주제방·프라이빗(관리자 개설).
Architecture

실시간, 그리고 비용 0

관리자 컨텐츠 — shared/mapContent · shared/news : 존·스팟·설정·지면, 전 사용자 실시간 반영(onSnapshot)
유저 컨텐츠 — liveFeed · liveSpots · liveRequests : 사진·좋아요·스팟·질문/응답, 계정 간 즉시 전파
안전장치 — 이미지 자동 압축(~170KB)·개수 캡·쿼리 제한 → Firestore 무료 티어 고정, 오프라인은 로컬 폴백
